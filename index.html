INTRODUCCIÓN

La limpieza de datos (en inglés data cleansing o data scrubbing) es el acto de descubrimiento y corrección o eliminación de registros de datos erróneos de una tabla o base de datos. El proceso de limpieza de datos permite identificar datos incompletos, incorrectos, inexactos, no pertinentes, etc. y luego substituir, modificar o eliminar estos datos sucios (“data duty”). Después de la limpieza, la base de datos podrá ser compatible con otras bases de datos similares en el sistema.

INTRODUCCIÓN

¿Por qué es importante la limpieza de datos?

La depuración de datos es vital para garantizar una alta integridad de los datos en cualquier investigación. Si toda la información con la que cuentas es confiable, entonces puedes estar seguro de que las decisiones que tomes con base en ella serán las más certeras.

Los datos de calidad pueden variar dependiendo de cuál sea su cualidad, entre las principales se encuentran:

  • Exactitud: todos los datos deben ser precisos. Una forma de comprobar su exactitud es comparándolos con otras fuentes. Si esta fuente no existe o es inexacta, entonces la información que tienes también lo será.
  • Coherencia: la coherencia de los datos permite saber si el dato sobre un objeto o persona es la misma en diferentes bases de datos, tablas o aplicaciones que utilices.
  • Validez: todos los datos deben cumplir con reglas o restricciones definidas. De igual forma, cada información debe poder ser validada para comprobar si son correctos o no.
  • Uniformidad: es importante que todos los datos dentro de tus bases tengan los mismos valores o unidades. Este es un elemento realmente indispensable a la hora de hacer data cleansing, pues de no tener todo en orden, el proceso se vuelve complejo.

    Quitar espacios y caracteres no imprimibles

    Cuando se trabajan con preguntas abiertas o variables de texto, es muy común que existan espacios en blanco al inicio y al final de la cadena de texto, o incluso entre los caracteres de la cadena de texto. Esto se produce por diferentes razones: incompatibilidad entre bases, errores en el input, en la transcripción, cambios de formato, etc. Tambien producto de todo esto se pueden generar los llamados caracteres no imprimibles, que “ensucian” el texto. Estos pueden ser: #!=$%():.,<>/ entre otros.

    Para esto usaremos la librería stringr para facilitarnos el trabajo

    #install.packages("stringr")   #Recordar que hay que instarlo primero
library(stringr)               #Cargamos la libreria

    Supongamos que tenemos la siguente cadena de texto con mucho espacio en blanco al inicio y al final

    s <- "      Hola, esta es un prueba        "
s
    [1] "      Hola, esta es un prueba        "
    str_trim(s)      #Elimina el espacio en blanco del inicio y del final de la cadena

    Hemos corregido el problema

    Quitar espacios y caracteres … continuación

    Para eliminar los caracteres no imprimibles, usaremos la función str_replace_all() que reemplaza todas las partes de una cadena que corresponden con un patrón con otro patrón. Por ejemplo, supongamos que tenemos la siguiente cadena de texto:

    cadena <- "Esta-es <la> ¡prueba! #1. (¿donde? +se= muestran), @los. /caracteres: $no% ¡imprimibles!, ¿es verdad?"
cadena
    [1] "Esta-es <la> ¡prueba! #1. (¿donde? +se= muestran), @los. /caracteres: $no% ¡imprimibles!, ¿es verdad?"

    De esta forma resolvemos el problema

    Ajustar valores fechas

    El trabajo con fechas siempre fue complejo desde el punto de vista del análisis de datos. Especialmente con los distintos formatos que se usan en el mundo, por ejemplo dd/mm/aaaa en Sudamérica, o mm/dd/aaaa en Estados Unidos por ejemplo. En R, cuando un campo fecha carga correctamente, lo transforma a un formato ISO 8601 aaaa-mm-dd. Sin embargo, ocurre que cuando cargamos bases de datos de diferentes orígenes a R, como .xlxs .csv .txt o de formatos como sql, spss u otros, la fecha llega con un formato numérico.

    Para transformar el campo “Fecha_nac” de un formato numérico a un formato de tipo fecha, vamos a usar la función as.Date() de R base, tomando como referencia la siguiente página

    #Transformar el campo Fecha_nac a tipo fecha
library(dplyr)  # Cragamos esta librería para poder usar la función mutate
fecha <- data_set['Fecha_nac'] %>%
  mutate(Fecha_nac_mod = as.Date(Fecha_nac,                # transformamos en Fecha_nac_mod
                                  origin = "1900-01-01",   # Fecha de origen para el conteo
                                  tz = "UTC"))             # Huso horario
data_set$Fecha_nac_mod <- fecha$Fecha_nac_mod #Integramos la nueva fecha a la base
